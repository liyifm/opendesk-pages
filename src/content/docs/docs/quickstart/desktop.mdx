## 1. 安装 OpenDesk Desktop

OpenDesk Desktop 提供图形化界面，无需安装 NodeJS 等运行环境。

### 1.1 下载安装包

从 [Release Notes](/news/) 页面获取对应平台的最新安装包：

| 平台 | 安装方式 |
|------|----------|
| **macOS** | 下载 `.dmg` 安装包，双击拖入 Applications |
| **Windows** | 下载 `.exe` 安装包，运行安装向导 |
| **HarmonyOS PC** | 下载 `.hap` 安装包，按系统提示安装 |

> 安装包和平台差异说明持续更新中，请关注 [Release Notes](/news/) 获取最新版本。

### 1.2 首次启动

安装完成后双击 OpenDesk 图标启动应用。首次启动时，先选择登录方式，再配置模型相关信息。

<img src="/assets/images/quickstart/desktop-login.png" alt="OpenDesk Desktop 登录方式配置" />

## 2. 快速完成初始配置

首次启动 OpenDesk Desktop 时，如果尚未配置模型，会直接弹出配置面板。面板左侧列出配置项，依次完成即可开始使用。

<img src="/assets/images/quickstart/desktop-login-model-init.png" alt="OpenDesk Desktop 初始模型设置" />

配置面板分为三个部分，从上到下依次操作：

### 模型服务商配置

点击「添加服务商」，选择你使用的模型服务商（如 OpenAI、硅基流动、智谱等），然后填入：

- **API 地址** — 服务商提供的 API Base URL
- **API Key** — 你的 API 密钥

> OpenDesk 支持所有提供 OpenAI 标准模型服务的厂商，填入对应的 API 地址和 Key 即可。

### 模型配置

在「模型配置」区域添加你想使用的具体模型：

- 输入模型的全名（如 `gpt-4o`、`claude-3-opus` 等）
- 为模型设置一个简短别名，方便日常切换

### 默认模型配置

在「默认模型配置」中选择刚才添加的模型作为默认模型，这样每次新建任务都会自动使用它。

## 3. 界面概览

启动后你会看到三栏布局：

<img src="/assets/images/quickstart/desktop-main.png" alt="OpenDesk Desktop 主界面" />

| 区域 | 功能 |
|------|------|
| **侧边栏** | 任务列表，支持搜索，底部为设置入口 |
| **对话区** | 核心交互区：任务标题栏 + 消息列表 + 输入框 |
| **结果区** | 右侧面板：展示生成的文件、执行步骤、文件变更；可展开/收起，也可导出为文件 |

Windows 上从开始菜单或搜索栏打开 OpenDesk 时，默认工作空间通常位于 `C:\Users\用户名\opendesk`。你也可以在桌面端切换工作空间；切换后，文件读写和任务上下文都会围绕新的工作空间展开。

### 设置与配置口令

如果后续需要重新修改模型配置，点击侧边栏底部的设置按钮，在设置页面中打开模型相关配置。

<img src="/assets/images/quickstart/desktop-model-setting.png" alt="OpenDesk Desktop 模型设置" />

OpenDesk 也支持一键快速导出口令。在设置页面进入「快捷操作」，然后使用「导出口令」或「导入口令」。口令中会包含基础配置，你的朋友在自己的 OpenDesk 中导入口令后，即可快速应用对应配置。

<img src="/assets/images/quickstart/desktop-quickpass.png" alt="OpenDesk Desktop 快捷操作口令" />

## 4. 创建你的第一个任务

用一句话描述需求即可。试试这样对它说：

- 「查看我今天的日程安排」
- 「帮我把这份销售数据做成分析报告」
- 「读一下这个项目的代码，帮我理清架构」

<img src="/assets/images/manual/quickstart_task.gif" alt="创建第一个任务" />

### 选择工作目录（可选）

1. 点击输入框正上方「当前工作空间」按钮
2. 选择当前任务要使用的目录
3. OpenDesk 仅在该目录内操作文件，不会影响其他位置

### 补充上下文（可选）

| 方式 | 说明 |
|------|------|
| 粘贴截图 | `Ctrl/Cmd + V` 粘贴剪贴板中的图片 |
| 上传文件 | 拖拽文件到输入框 |
| 补充说明 | 明确目标、范围、约束和预期输出 |

## 5. 查看结果与追问

任务完成后，右侧结果区包含三类视图：

| 视图 | 内容 |
|------|------|
| **产物视图** | 任务生成的文件（文档、数据、图片等） |
| **变更视图** | 文件的新增/删除/修改（diff 高亮对比） |
| **记忆视图** | 任务过程中自动维护的结构化记忆 |

追问无需重复背景，直接说改什么即可：

- 「再加一个饼图展示占比」
- 「导出为 PDF」

## 6. 应用与技能中心

桌面版在 Agent 能力之外，还提供了一组图形化内置应用，用于降低常见场景的使用门槛。你可以通过侧边栏入口打开应用，也可以直接把目标告诉 OpenDesk，让它在需要时调用对应能力。

<img src="/assets/images/quickstart/desktop-applications.png" alt="OpenDesk Desktop 应用入口" />

点击侧边栏的技能快捷图标，可以打开技能中心。你可以搜索需要的技能，也可以直接选中推荐技能并安装。

<img src="/assets/images/quickstart/desktop-skills-center.png" alt="OpenDesk Desktop 技能中心" />
